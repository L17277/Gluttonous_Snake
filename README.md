🐍 项目概述：智能贪吃蛇系统（AI训练 + 图形控制台）
本项目是一个功能完整的贪吃蛇游戏 + 强化学习训练平台，包含三个核心模块：游戏引擎（snake_game.py）、图形启动器（launcher.py）和深度强化学习训练脚本（ai.py）。用户既可以通过启动器进行手动游戏，也可以启动AI训练，所有参数（如速度、地图大小、特殊食物机制、奖励函数）均可实时调节，并支持游戏记录与回放。

1. 游戏核心（snake_game.py）
基础机制：经典贪吃蛇规则，蛇移动、吃红点增长、撞墙或自碰即死。额外提供“穿墙模式”，蛇头可从对侧出现。

特殊食物系统：

黄点：当得分为6的倍数时出现（吃黄点达到的倍数不触发），持续6秒。蛇头吃到时，根据剩余时间（1~6秒）获得 (6,5,4,3,2,1) × 基础分 的分数，不增加长度。可独立开关，并调节触发倍数、持续时间和基础分。

蓝点：当得分为10的倍数时出现，持续5秒。蛇头吃到时，根据剩余时间（1~3秒）减少 (3,2,1) × 减长倍数 节身体，不改变分数。可独立开关，并调节触发倍数、持续时间和减长倍数。

无论黄点或蓝点是否过期，只要蛇头触碰即消失（过期后不产生效果）。

日志与回放：手动游戏时可记录每一帧的状态（蛇身坐标、红点位置、黄/蓝点位置及剩余时间、分数、方向），保存为JSON格式。回放时自动读取日志中的地图大小，逐帧还原游戏过程。

AI 环境接口：GameEnv 类为强化学习提供标准 reset / step 接口。状态以结构化格子列表返回，每个元素包含：

type：蛇身（含距离头部长度）、红点、黄点（含剩余时间）、蓝点（含剩余时间）

pos：坐标

dist_from_head：仅蛇身，表示该节在身体中的位置

remaining：特殊食物剩余秒数

手动操控：支持方向键和 WASD，采用 pygame.key.get_pressed() 轮询，优先级固定（上>下>左>右），避免同时按键导致意外转向。

2. 图形启动器（launcher.py）
基于 Tkinter 构建的图形界面，提供完整的控制面板：

基础设置：速度（帧率）、地图大小、边界模式（死亡/穿墙）

黄点/蓝点独立控制：分别开关，并可调整触发倍数、持续时间、基础分/减长倍数

奖励设计：AI训练专用，可独立调节红点奖励、每步惩罚、死亡惩罚（手动模式忽略这些值）

运行模式：切换“手动游戏”或“AI 训练（可视化）”

记录选项：手动游戏时是否保存日志（自动生成时间戳文件）

操作按钮：启动、停止进程、回放日志（选择JSON文件后启动回放窗口）

历史得分：自动记录最近20条得分（带时间戳），存储于 scores.txt

启动器将用户界面的所有参数通过命令行参数传递给 snake_game.py（手动模式）或 ai.py（AI模式），实现参数同步。

3. AI 训练脚本（ai.py）
采用 DQN + Transformer 编码器 架构，利用 GameEnv 提供的结构化状态进行决策：

模型：SnakeTransformer 将每个格子信息（类型、坐标、距离、剩余时间）分别嵌入为向量，求和后送入 Transformer 编码器，再对序列维度池化得到固定长度特征，最后输出四个动作（上、下、左、右）的 Q 值。

状态处理：由于状态为变长列表，训练时使用 pad_sequence 和 src_key_padding_mask 进行批处理。

训练算法：经验回放（Replay Buffer）、目标网络（每10局更新）、ε-贪心探索（起始1.0，按0.995衰减至0.01）。

奖励机制：与启动器中设置的 reward_red、reward_step、reward_death 完全一致，黄点奖励等于所获得的分数（多倍基础分），蓝点奖励为0。

可视化：训练时显示游戏窗口，便于观察行为。

模型保存：第1局及每10局自动保存模型权重（文件名为 dqn_snake_epXXX.pth），训练结束时保存最终模型。

4. 项目特色
高度可配置：所有游戏参数、特殊食物参数、奖励参数均可通过启动器调节，无需修改代码。

结构化状态：AI 获得每个格子的细粒度信息，而非简单向量，利于学习复杂策略。

日志与回放：完整的记录系统，可用于分析游戏过程或复现精彩对局。

穿墙与特殊食物：提供变体规则，增加游戏趣味性。

跨平台：基于 Python + Pygame + Tkinter + PyTorch，支持 Windows / Linux / macOS。

5. 运行流程
安装依赖：pygame（必需）、torch 和 numpy（AI训练）。

启动 launcher.py，调节参数并选择模式。

点击“启动”：手动模式打开游戏窗口；AI模式开始训练并弹出游戏窗口（可视化）。

手动游戏可随时退出，得分自动记录；AI训练过程中控制台输出每局得分、平均分、步数、epsilon，同时模型定期保存。

手动游戏若勾选“记录日志”，生成的 JSON 文件位于 logs/ 目录，可随时通过“回放日志”按钮重放。

6. 技术栈
界面：Tkinter（启动器）

游戏渲染：Pygame

深度学习：PyTorch（Transformer、DQN）

数据处理：NumPy

日志格式：JSON（可读性强，便于分析）

此项目不仅是一个完整的贪吃蛇游戏，更是一个灵活的实验平台，适用于探索强化学习在简单环境中的应用。
